Webhook endpoint without signature verification

The /stripe-webhook endpoint processes incoming webhook events without verifying the Stripe-Signature header. Attackers can forge webhook events to mark unpaid orders as paid, confirm fraudulent transactions, or trigger refund operations.

Payment operation without idempotency key

stripe.Charge.create() is called without an idempotency_key parameter. Network retries or user double-clicks can create duplicate charges, leading to customer complaints and potential chargebacks.

app/api/charges.py:45 — CWE-799 · A04
HIGH

Payment amount from client without server validation

The payment amount is read directly from request.json.get('amount') and passed to the payment gateway. An attacker can modify the amount in transit (e.g., change $99.99 to $0.01) using browser dev tools or a proxy.
